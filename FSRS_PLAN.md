# План разработки FSRS плагина для Obsidian

## Обзор
FSRS (Free Spaced Repetition Scheduler) — это алгоритм интервального повторения, основанный на машинном обучении. Данный плагин интегрирует FSRS в Obsidian для управления карточками для запоминания непосредственно в заметках.

## Текущее состояние (Done ✅)

### ✅ Базовая интеграция WASM
- **WASM модуль на Rust**: Используется библиотека `rs-fsrs` версии 1.2.1
- **Компиляция WASM**: Настроены скрипты `build-wasm`, `encode-wasm` в `package.json`
- **Base64 кодирование**: WASM модуль кодируется в base64 для встраивания в TypeScript код
- **Инициализация WASM**: Корректная загрузка и инициализация WASM модуля из base64 строки

### ✅ Основной функционал плагина
- **Команда "Добавить поля FSRS"**: Добавляет YAML frontmatter с полями FSRS в текущий файл
- **Команда "Найти карточки для повторения"**: Сканирует все файлы и находит карточки, у которых наступила дата повторения
- **Команда "Повторить текущую карточку"**: Обновляет карточку после ответа (пока с фиксированной оценкой "Good")
- **Настройки плагина**: Базовые настройки через вкладку Settings (пока минимальные)

### ✅ WASM функции (реализованы в Rust)
- `get_fsrs_yaml()` — возвращает YAML строку со всеми полями FSRS для новой карточки
- `create_fsrs_card_json()` — создает JSON представление новой карточки
- `review_card()` — обновляет карточку на основе оценки пользователя
- `get_fsrs_yaml_after_review()` — возвращает обновленный YAML после повторения
- `get_next_review_dates()` — возвращает возможные даты следующего повторения для всех оценок
- `is_card_due()` — проверяет, готова ли карточка к повторению
- `get_retrievability()` — вычисляет извлекаемость (retrievability) карточки
- `card_state_to_string()` — конвертирует состояние карточки в строку
- `get_current_time()` — возвращает текущее время в формате ISO 8601

### ✅ Структура полей FSRS в YAML
Поля сохраняются в frontmatter заметки:
```yaml
fsrs_due: "2024-01-01T00:00:00Z"
fsrs_stability: 0.0
fsrs_difficulty: 0.0
fsrs_elapsed_days: 0
fsrs_scheduled_days: 0
fsrs_reps: 0
fsrs_lapses: 0
fsrs_state: "New"
fsrs_last_review: "2024-01-01T00:00:00Z"
```

### Управление WASM
- **Проблема**: Большой base64 файл (~236KB) занимает много контекста
- **Решение**: Хранить WASM в отдельном файле и загружать динамически
- **Альтернатива**: Использовать `esbuild-plugin-wasm` для встраивания WASM в сборку
