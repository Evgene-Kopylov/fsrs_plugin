# Новый формат YAML для FSRS плагина Obsidian

## Обзор

Новый формат YAML для карточек FSRS значительно упрощает хранение данных, вынося алгоритмические параметры в настройки плагина и сохраняя в каждой карточке только историю повторений.

### Основные изменения

| Старый формат | Новый формат |
|---------------|--------------|
| Хранил все параметры алгоритма в каждой карточке | Хранит только историю сессий |
| Жестко закодированные веса алгоритма | Использует фиксированные веса из библиотеки `rs-fsrs` |
| Сложная миграция при обновлении алгоритма | Простое обновление через настройки плагина |
| Нет истории повторений | Полная история всех сессий повторения |

## Формат карточки

### Структура YAML

```yaml
---
reviews:
  - date: 2025-05-14T10:00:00
    rating: Good
    stability: 6.5
    difficulty: 3.2
  - date: 2025-05-07T09:00:00
    rating: Easy
    stability: 3.2
    difficulty: 3.0
---
```

### Поля карточки

#### Обязательные поля:
- **`reviews`** - массив сессий повторения (может быть пустым для новых карточек). Наличие этого поля определяет, является ли файл карточкой FSRS.

#### Поля сессии:
- **`date`** - дата и время сессии в формате ISO 8601 (`YYYY-MM-DDTHH:MM:SS`)
- **`rating`** - оценка повторения: `Again`, `Hard`, `Good`, `Easy`
- **`stability`** - стабильность карточки после сессии (число)
- **`difficulty`** - сложность карточки после сессии (число)

## Вычисляемые поля

Следующие поля **не хранятся** в YAML, а вычисляются на основе последней сессии и параметров алгоритма:

### Текущее состояние:
- **`due`** - следующая дата повторения (вычисляется алгоритмом FSRS)
- **`stability`** - текущая стабильность (= последняя сессия.stability)
- **`difficulty`** - текущая сложность (= последняя сессия.difficulty)
- **`state`** - текущее состояние карточки: `New`, `Learning`, `Review`, `Relearning`
- **`retrievability`** - извлекаемость (вероятность запоминания, 0.0-1.0)

### Статистика:
- **`elapsed_days`** - дней с последнего повторения
- **`scheduled_days`** - запланированных дней до следующего повторения
- **`reps`** - общее количество повторений
- **`lapses`** - количество ошибок (оценок `Again`)

## Настройки плагина

Параметры алгоритма теперь хранятся в настройках плагина (**Settings → FSRS Algorithm Settings**):

### Параметры алгоритма:
- **Request Retention** (0.5-1.0) - целевой уровень запоминания
- **Maximum Interval** (дни) - максимальный интервал между повторениями
- **Enable Interval Fuzz** - случайное изменение интервалов (±5%)

### Настройки по умолчанию:
- **Initial Stability** - начальная стабильность для новых карточек
- **Initial Difficulty** - начальная сложность для новых карточек

### Важное ограничение:
Библиотека `rs-fsrs` использует **фиксированные веса алгоритма**, которые нельзя изменять через настройки плагина. Для настройки весов используйте библиотеку `fsrs-rs`.

## Использование

### Создание новой карточки:
1. Откройте файл в Obsidian
2. Выполните команду **"Добавить поля FSRS в шапку файла"**
3. В frontmatter добавится пустой массив `reviews: []`

### Повторение карточки:
1. Откройте файл с карточкой
2. Выполните команду **"Повторить текущую карточку"**
3. Выберите оценку в модальном окне:
   - **Again (1)** - забыл
   - **Hard (2)** - вспомнил с трудом
   - **Good (3)** - вспомнил
   - **Easy (4)** - вспомнил легко

### Просмотр карточек для повторения:
1. В любом файле добавьте блок кода:
   ````markdown
   ```fsrs-now
   ```
   ````
2. Плагин автоматически отобразит все просроченные карточки

## Миграция со старого формата

### Для новых пользователей:
1. Удалите старые тестовые карточки
2. Создайте новые карточки с помощью команды **"Добавить поля FSRS"**

### Для существующих пользователей (если понадобится):
1. Старый формат:
```yaml
fsrs_due: "2024-01-01T00:00:00Z"
fsrs_stability: 0.0
fsrs_difficulty: 0.0
fsrs_elapsed_days: 0
fsrs_scheduled_days: 0
fsrs_reps: 0
fsrs_lapses: 0
fsrs_state: "New"
fsrs_last_review: "2024-01-01T00:00:00Z"
```

2. Новый формат (эквивалент):
```yaml
reviews:
  - date: 2024-01-01T00:00:00Z
    rating: Good
    stability: 0.0
    difficulty: 0.0
```

## Преимущества нового формата

### 🚀 **Упрощение обновлений**
- Изменение параметров алгоритма в настройках применяется ко всем карточкам
- Не нужно редактировать каждую карточку отдельно

### 📊 **Полная история**
- Сохраняется каждая сессия повторения
- Возможность анализа прогресса обучения
- Восстановление состояния из истории

### 🔧 **Гибкость**
- Разные типы карточек могут использовать разные профили настроек (в будущем)
- Легкое добавление новых параметров алгоритма

### 🏗️ **Чистая архитектура**
- Разделение данных: хранение vs вычисление
- Меньше дублирования в хранилище
- Более эффективное использование пространства

### ✅ **Упрощенная логика**
- Нет избыточного флага `srs: true`
- Участие в SRS определяется наличием поля `reviews`
- Меньше проверок в коде

## Ограничения

### Фиксированные веса алгоритма:
Библиотека `rs-fsrs` версии 1.2.1, используемая в плагине, имеет предобученные веса алгоритма. Эти веса нельзя изменить через настройки плагина. Для кастомизации весов используйте библиотеку `fsrs-rs`.

### Производительность:
- При первом открытии файла с блоком `fsrs-now` происходит сканирование всего хранилища
- Для больших хранилищ (>1000 файлов) это может занять 2-3 секунды
