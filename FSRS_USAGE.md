# Использование блока fsrs-now в FSRS плагине для Obsidian

## 📖 Введение

Блок `fsrs-now` — это **динамически рендерируемый** список карточек FSRS, которые готовы к повторению. Вместо статического текста в блоке кода, плагин автоматически рендерит интерактивный интерфейс прямо на месте блока, аналогично тому, как это делает Journals с `calendar-timeline`.

### Основные принципы новой архитектуры:

1. **Пустой блок кода**: Вы добавляете только ````fsrs-now```` (без содержимого)
2. **Динамический рендеринг**: При открытии файла плагин автоматически загружает и отображает карточки
3. **Интерактивность**: Кликабельные ссылки, кнопки действий, автоматическое обновление
4. **CSS стилизация**: Профессиональный внешний вид с цветовым кодированием состояний

## 🚀 Как использовать

### 1. Добавить пустой блок fsrs-now в файл

Просто добавьте в любое место вашего markdown-файла:

````markdown
```fsrs-now
```
````

### 2. Автоматический рендеринг

При открытии файла плагин автоматически:
1. Просканирует всё ваше хранилище на наличие карточек FSRS
2. Определит, какие карточки просрочены (дата повторения `fsrs_due` наступила)
3. Отрендерит интерактивный список на месте блока

### 3. Где использовать блок

Вы можете добавить блок `fsrs-now` в любой markdown-файл:
- **Ежедневные заметки**: Для ежедневного повторения
- **Файл повторений**: Например, `FSRS Review.md`
- **Панель управления**: Создайте отдельный файл как панель управления карточками
- **Статистика**: Добавьте блок в файл с вашей статистикой обучения

## 🧭 Навигация и интерактивность

### Кликабельные элементы:

1. **Ссылки на файлы**: Имя файла является кликабельной ссылкой для перехода к карточке
2. **Кнопка "Открыть"**: Открывает карточку в новой вкладке Obsidian
3. **Кнопка "Повторить"**: Открывает карточку с фокусом на повторении
4. **Кнопка "Обновить"**: Перезагружает список карточек без перезагрузки страницы

### Сортировка карточек:

Карточки автоматически сортируются:
- **По дате повторения**: Самые старые первыми (максимальная просрочка)
- **По извлекаемости**: Самые забытые карточки получают приоритет

### Лимит отображения:

По умолчанию отображается **первые 30 карточек**:
- Показаны первые 30 карточек с наибольшей просрочкой
- Внизу указывается количество скрытых карточек
- Например: `Показано: 30 из 45 карточек (15 скрыто)`

## 🎨 Стили и кастомизация

### Цветовое кодирование состояний:

Каждое состояние карточки имеет свой цвет:

- **New (Новая)**: Серый (`border-left: 3px solid var(--text-muted)`)
- **Learning (Изучается)**: Синий (`border-left: 3px solid var(--color-blue)`)
- **Review (Повторение)**: Зелёный (`border-left: 3px solid var(--color-green)`)
- **Relearning (Переучивание)**: Оранжевый (`border-left: 3px solid var(--color-orange)`)

## 🔧 Техническая реализация

### Архитектура плагина:

1. **`registerMarkdownCodeBlockProcessor`**:
   - Регистрирует обработчик для блоков `fsrs-now`
   - Автоматически вызывается при рендеринге markdown
   - Передает управление классу `FsrsNowRenderer`

2. **Класс `FsrsNowRenderer`**:
   - Наследуется от `MarkdownRenderChild`
   - Динамически загружает карточки через `plugin.getCardsForReview()`
   - Рендерит HTML интерфейс с обработчиками событий
   - Управляет жизненным циклом компонента

3. **Метод `getCardsForReview()`**:
   - Сканирует все markdown файлы хранилища
   - Парсит frontmatter с полями FSRS
   - Фильтрует карточки с просроченной датой `fsrs_due`
   - Возвращает отсортированный массив карточек

### Процесс рендеринга:

```typescript
1. Пользователь открывает файл с блоком ```fsrs-now```
2. Obsidian вызывает registerMarkdownCodeBlockProcessor
3. Создается экземпляр FsrsNowRenderer
4. Рендерер показывает индикатор загрузки
5. Асинхронно загружаются карточки для повторения
6. Динамически генерируется HTML интерфейс
7. Добавляются обработчики событий для кнопок и ссылок
```

## 🔄 Обновление списка

### Автоматическое обновление:

1. **При открытии файла**: Список всегда актуален
2. **Кнопка "Обновить"**: Мгновенное обновление без перезагрузки страницы
3. **Команда плагина**: "Найти карточки для повторения"

### Как работает обновление:

```typescript
// При нажатии кнопки "Обновить"
async refresh() {
    // Показываем индикатор загрузки
    this.container.innerHTML = `<div class="fsrs-now-loading">...</div>`;
    
    // Заново загружаем карточки
    const cardsForReview = await this.plugin.getCardsForReview();
    
    // Перерендериваем интерфейс
    this.renderCards(cardsForReview);
}
```

## 🚨 Решение проблем

### Проблема: "Блок fsrs-now пустой"
**Решение**: 
1. Проверьте, что у вас есть карточки с полями FSRS в frontmatter
2. Убедитесь, что дата `fsrs_due` наступила
3. Выполните команду "Найти карточки для повторения" для принудительного обновления

### Проблема: "Кнопки не работают"
**Решение**:
1. Перезагрузите Obsidian
2. Проверьте, включен ли плагин в настройках
3. Убедитесь, что WASM модуль загружен (проверьте консоль разработчика)

### Проблема: "Медленная загрузка"
**Решение**:
1. Плагин сканирует все файлы хранилища - это может занять время
2. Для больших хранилищ используйте блок только в нужных файлах
3. В будущих версиях планируется кэширование результатов

## 💡 Советы и рекомендации

### Оптимальное использование:

1. **Ежедневный ритуал**: Добавьте блок в ваш daily note template
2. **Фокусные сессии**: Создайте отдельный файл для каждой темы/предмета
3. **Мониторинг прогресса**: Добавьте блок в файл статистики и обновляйте раз в неделю
4. **Командная работа**: Используйте общий файл для командного обучения

### Интеграция с другими плагинами:

- **Dataview**: Экспортируйте данные для сложного анализа
- **Tasks**: Создавайте задачи на основе карточек для повторения
- **Calendar**: Планируйте повторения в календаре
- **Templater**: Автоматически добавляйте блок в новые файлы

### Производительность:

1. **Размер хранилища**: Для хранилищ > 1000 файлов первое сканирование может занять 2-3 секунды
2. **Кэширование**: Результаты не кэшируются между сессиями (планируется)
3. **WASM инициализация**: Загрузка WASM модуля происходит один раз при старте плагина

## 🎯 Быстрый старт

### За 5 минут:

1. **Установите плагин**: Скопируйте `main.js`, `manifest.json`, `styles.css` в папку плагина
2. **Включите плагин**: Settings → Community plugins → FSRS Plugin
3. **Добавьте блок**: Создайте файл и добавьте ````fsrs-now````
4. **Создайте карточки**: Используйте команду "Добавить поля FSRS в шапку файла"
5. **Начните повторять**: Откройте файл с блоком и кликайте на карточки
