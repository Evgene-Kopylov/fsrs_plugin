
# План миграции TypeScript кода в Rust для плагина FSRS (черновик)

## Текущее состояние

### Уже реализовано в Rust/WASM:
- ✅ Основные алгоритмы FSRS (`review_card`, `compute_current_state`)
- ✅ Парсинг YAML/frontmatter (`extract_fsrs_from_frontmatter_wrapped`)
- ✅ Создание YAML для карточек
- ✅ Фильтрация, сортировка и группировка карточек
- ✅ Работа со временем (`get_overdue_hours`, `is_card_overdue`)

### Остается в TypeScript (нельзя перенести):
- ❌ Интеграция с Obsidian API (чтение/запись файлов, UI)
- ❌ Кэширование (зависит от среды выполнения)
- ❌ Команды плагина и UI компоненты
- ❌ Настройки плагина (сериализация)

## План миграции по фазам

### Фаза 1: Улучшение парсинга (1-2 дня)

**Новые функции Rust:**
```rust
// Проверяет содержимое файла на признаки FSRS
#[wasm_bindgen]
pub fn should_process_file_content(content: String) -> bool

// Извлекает все карточки FSRS из массива файлов
#[wasm_bindgen]
pub fn batch_extract_fsrs_from_contents(
    files_json: String, // [{path: "path.md", content: "..."}, ...]
    ignore_patterns_json: String // массив паттернов
) -> String // [{success: bool, card: {...}, error: string}, ...]
```

**Изменения TypeScript:**
- Обновить `shouldProcessFile` для вызова Rust функции
- Обновить `performFullScan` для использования batch извлечения

### Фаза 2: Batch операции (2-3 дня)

**Новые функции Rust:**
```rust
// Массовое вычисление состояний карточек
#[wasm_bindgen]
pub fn batch_compute_states(
    cards_json: String, // массив карточек
    settings_json: String,
    now_iso: String
) -> String // массив ComputedCardState

// Массовая фильтрация карточек для повторения
#[wasm_bindgen]
pub fn batch_filter_due_cards(
    cards_json: String,
    settings_json: String,
    now_iso: String
) -> String // массив отфильтрованных карточек
```

**Изменения TypeScript:**
- Рефакторинг `getCardsForReview`: передавать данные в Rust для batch обработки
- Уменьшение количества вызовов WASM с O(n) до O(1)

## Ожидаемые преимущества

### Производительность:
- **Сокращение вызовов WASM** с O(n) до O(1) для batch операций
- **Более эффективная обработка данных** в Rust (компилируемый язык)
- **Уменьшение накладных расходов** на сериализацию/десериализацию

### Удобство разработки:
- **Единая кодовая база** для алгоритмов FSRS
- **Более строгая типизация** в Rust предотвращает ошибки времени выполнения
- **Лучшая тестируемость** алгоритмов вне контекста Obsidian

### Поддерживаемость:
- **Четкое разделение ответственности**: TypeScript - интеграция, Rust - логика
- **Упрощенный рефакторинг** алгоритмов FSRS
- **Возможность reuse** Rust кода в других проектах

## Риски и ограничения

### Технические ограничения:
- WASM не имеет прямого доступа к файловой системе Obsidian
- Ограничения на передачу больших объемов данных через WASM boundary
- Сложность отладки кода, работающего в обеих средах

### Меры предосторожности:
1. Сохранять обратную совместимость на каждом этапе
2. Добавлять подробное логирование для отладки
3. Тестировать производительность после каждого изменения
4. Поддерживать fallback на TypeScript реализации при ошибках WASM
