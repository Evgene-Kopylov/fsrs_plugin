## Задание для агента №1: Создание основы локализации и перевод модального окна повторения

### Цель
Создать инфраструктуру для локализации плагина и перевести **только модальное окно повторения** (`ReviewModal`) на английский язык (по умолчанию) с возможностью добавления русского перевода.

### Контекст
В настоящее время все строки в плагине жёстко зашиты на русском языке. Необходимо внедрить систему локализации, начиная с одного компонента, чтобы в будущем легко перевести остальные части.

### Задачи

#### 1. Создать структуру файлов локализации
- Создать папку `src/locales/`
- Внутри создать два файла:
  - `en.json` – английские строки
  - `ru.json` – русские строки

#### 3. Создать менеджер локализации
Создать файл `src/utils/i18n.ts` со следующим содержимым:

```typescript
import en from '../locales/en.json';
import ru from '../locales/ru.json';

class I18n {
  private currentLocale: string;
  private translations: Record<string, any>;

  constructor() {
    // Пока язык по умолчанию – английский (позже добавим настройку)
    this.currentLocale = 'en';
    this.loadTranslations();
  }

  private loadTranslations() {
    if (this.currentLocale === 'ru') {
      this.translations = ru;
    } else {
      this.translations = en;
    }
  }

  public setLocale(locale: 'en' | 'ru') {
    this.currentLocale = locale;
    this.loadTranslations();
  }

  public t(key: string, params?: Record<string, string | number>): string {
    const keys = key.split('.');
    let value: any = this.translations;
    for (const k of keys) {
      if (value && typeof value === 'object' && k in value) {
        value = value[k];
      } else {
        console.warn(`Missing translation key: ${key}`);
        return key;
      }
    }
    if (typeof value !== 'string') {
      return key;
    }
    if (params) {
      return value.replace(/\{(\w+)\}/g, (_, param) => String(params[param] ?? `{${param}}`));
    }
    return value;
  }
}

export const i18n = new I18n();
```

#### 4. Модифицировать `ReviewModal` для использования локализации
Файл: `src/commands/review/review-modal.ts`

Требуется полностью переписать метод `onOpen()`, заменив все жёстко заданные строки на вызовы `i18n.t()`. При этом:

- Заголовок модального окна: `contentEl.createEl("h3", { text: i18n.t('review.title') });`
- Информационный блок: создать элементы через `createEl`, установить текст через `textContent` и `createEl` с текстом (не использовать `innerHTML`).
- Кнопки оценок: текст кнопок брать из `i18n.t('review.buttons.again')` и т.д.
- Кнопка "Отмена": текст из `i18n.t('review.buttons.cancel')`.

**Важно:** При формировании информационного блока не использовать `innerHTML`. Вместо этого создавать элементы:

```typescript
const info = contentEl.createEl('div');
const small = info.createEl('small');

// Файл
small.createEl('strong', { text: i18n.t('review.file_label') });
small.appendText(' ' + this.card.filePath);
small.createEl('br');

// Сессии
small.createEl('strong', { text: i18n.t('review.sessions_label') });
small.appendText(' ' + this.card.reviews.length);
small.createEl('br');

// Последняя
small.createEl('strong', { text: i18n.t('review.last_review_label') });
const lastReviewStr = this.card.reviews.length > 0 
  ? new Date(this.card.reviews[this.card.reviews.length - 1].date).toLocaleString()
  : i18n.t('review.buttons.cancel'); // или "none" – позже добавим отдельный ключ
small.appendText(' ' + lastReviewStr);
small.createEl('hr');
```

- Для цветов кнопок оставить как есть, менять только текст.

#### 5. Временно не трогать уведомления (Notices) в этом компоненте
В `review-modal.ts` нет уведомлений, только UI. Поэтому дополнительные изменения не требуются. Уведомления будут переведены на следующем этапе.

### Критерии выполнения
- Созданы файлы `en.json`, `ru.json` в `src/locales/`.
- Создан файл `i18n.ts` с корректной типизацией.
- В `ReviewModal` все текстовые строки заменены на вызовы `i18n.t()`, нигде не используется `innerHTML`.
- Модальное окно открывается, отображает английские заголовки и текст кнопок.
- При смене языка в коде (вызов `i18n.setLocale('ru')`) – тексты должны стать русскими (пока без UI переключения, просто проверка вручную).

### Проверка
- Запустить плагин в Obsidian.
- Вызвать повторение карточки (команда "Повторить текущую карточку").
- Убедиться, что заголовок, подписи и кнопки отображаются на английском языке.
- Временно изменить в `i18n.ts` `this.currentLocale = 'ru'`, перезагрузить плагин, проверить русский язык.

### Результат
Плагин после изменений:
- Имеет основу для локализации.
- Модальное окно повторения полностью переведено.
- Код безопасен (нет `innerHTML`).
- Остальные части плагина пока не затронуты (работают как раньше, на русском).

### Примечание для агента
- Не изменяйте другие файлы, кроме указанных.
- Не удаляйте существующие русские строки из других частей плагина – они будут заменены позже.
- Следите за тем, чтобы импорты JSON были корректны (возможно, потребуется настроить `tsconfig.json` для разрешения импорта JSON – добавьте `"resolveJsonModule": true` в `compilerOptions`). Если такого нет, добавьте его.
