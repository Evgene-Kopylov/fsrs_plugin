## Задание для агента №2: Перевод уведомлений и основных команд плагина

### Цель
Перевести все пользовательские уведомления (`new Notice(...)`) и тексты команд (названия команд в Obsidian) на английский язык с использованием созданной системы локализации. Сохранить возможность отображения русского языка при переключении локали.

### Контекст
В плагине есть множество вызовов `new Notice()` с русскими текстами (например, «Нет активного файла», «Файл не содержит frontmatter», «Карточка повторена: Good» и т.д.). Также названия команд, которые видны в палитре команд Obsidian, сейчас на русском. Их необходимо перевести на английский (базовый язык) и добавить русские варианты в `ru.json`.

### Задачи

#### 2. Создать вспомогательную функцию для уведомлений
В файле `src/utils/i18n.ts` добавить функцию `notice`:

```typescript
import { Notice } from 'obsidian';

export function showNotice(key: string, params?: Record<string, string | number>): void {
  new Notice(i18n.t(key, params));
}
```

Экспортировать её из модуля.

#### 3. Заменить все вызовы `new Notice(...)` в следующих файлах:

**`src/commands/add-fsrs-fields.ts`**:
- `new Notice("Нет активного файла")` → `showNotice('notices.no_active_file')`
- `new Notice("В файле уже есть поля FSRS...")` → `showNotice('notices.fsrs_fields_exists')`
- `new Notice("Поля FSRS (новый формат) добавлены в файл")` → `showNotice('notices.fsrs_fields_added')`
- `new Notice("Ошибка при добавлении полей FSRS: " + errorMessage)` → `showNotice('notices.error_parsing_card', { error: errorMessage })`

**`src/commands/find-fsrs-cards.ts`**:
- `new Notice("Нет карточек для повторения")` → `showNotice('notices.no_cards_for_review')`
- `new Notice("Найдено ${cardsForReview.length} карточек...")` → `showNotice('notices.cards_found', { count: cardsForReview.length })`
- `new Notice("Нет активного файла для вставки блока кода")` → `showNotice('notices.no_active_file')`
- `new Notice("Добавлен блок fsrs-table с ${cardsForReview.length} карточками...")` → `showNotice('notices.fsrs_table_inserted', { count: cardsForReview.length })`
- Ошибки аналогично.

**`src/commands/review/review-card.ts`** и **`reviewCardByPath`**:

#### 4. Перевести названия команд
В `src/commands/index.ts` заменить русские названия в `addCommand` на вызовы `i18n.t('commands.xxx')`:

```typescript
plugin.addCommand({
  id: "add-fsrs-fields",
  name: i18n.t('commands.add_fsrs_fields'),
  callback: async () => { ... }
});
// и так для всех шести команд
```

#### 5. Временно добавить переключение языка для тестирования
В `main.ts` после загрузки настроек установить локаль:

```typescript
import { i18n } from './utils/i18n';

async loadSettings() {
  this.settings = Object.assign({}, DEFAULT_SETTINGS, await this.loadData());
  i18n.setLocale(this.settings.language || 'en');
}
```

В настройки пока не добавлять UI, просто использовать значение по умолчанию 'en' (или из настроек, если поле `language` уже есть). Если поля нет – добавить в `types.ts`:

```typescript
language: 'en',
```

### Критерии выполнения
- Все вызовы `new Notice(...)` в перечисленных файлах заменены на `showNotice(...)` с корректными ключами.
- Файлы локализации дополнены новыми ключами.
- Названия команд отображаются на английском языке в палитре команд (при локали 'en') и на русском при смене локали на 'ru'.
- Уведомления при работе плагина выводят корректные сообщения на выбранном языке.
- Параметризованные сообщения (например, `{count}`, `{rating}`) корректно подставляют значения.
- Старые русские строки удалены из кода (заменены на вызовы локализации).

### Проверка
- Запустить плагин. Открыть палитру команд (`Ctrl/Cmd+P`) – увидеть команды на английском.
- Вызвать команду "Add FSRS fields to frontmatter" – должно появиться уведомление "FSRS fields added to file".
- Попытаться добавить поля повторно – уведомление "File already has FSRS fields...".
- Найти карточки для повторения – уведомление с количеством.
- Повторить карточку – уведомление "Card reviewed: Good" и т.д.
- Удалить последнее повторение – уведомление "Last review deleted".
- Временно изменить в коде `i18n.setLocale('ru')` – все уведомления и команды должны стать русскими.

### Результат
Плагин после изменений:
- Все уведомления и команды локализованы.
- Базовая система локализации работает.
- Код чище, без жёстко зашитых строк.
- Готов к добавлению перевода остальных компонентов (статус-бар, таблица, справка) на следующих этапах.

### Примечание для агента
- Не забудьте импортировать `showNotice` в каждом файле, где она используется.
- Для сообщений об ошибках, где есть динамический `errorMessage`, используйте параметр `{error}`.
- Для уведомления о досрочном повторении (сложное сообщение) – добавьте новый ключ `card_not_due_with_early` и используйте его. Если останутся другие сложные сообщения, создавайте новые ключи по мере необходимости.
- Убедитесь, что после замены код компилируется без ошибок.
