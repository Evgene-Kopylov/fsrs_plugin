## Задание для агента №4: Локализация таблицы `fsrs-table`

### Цель
Перевести все текстовые элементы, связанные с отображением и работой блока `fsrs-table`:
- Заголовки колонок по умолчанию (например, "file", "reps", "overdue", "state", "due").
- Сообщения об ошибках (например, "FSRS: Error:\n-- PARSING FAILED...").
- Сообщения при пустом результате (нет карточек).
- Информационные сообщения (например, "Показано: X из Y карточек (Z скрыто)").
- Сообщения при загрузке ("Loading FSRS cards...").

### Контекст
Файлы, которые нужно модифицировать:

- `src/ui/fsrs-table-renderer.ts` – содержит сообщение "Loading FSRS cards..." и `renderEmptyState()` (сейчас пусто, но нужно добавить сообщение).
- `src/utils/fsrs-table-params.ts` – содержит `formatError()` (используется при парсинге SQL).
- `src/ui/fsrs-help-modal.ts` – не входит в это задание, будет отдельно.

#### 2. Модифицировать `src/utils/fsrs-table-format.ts`

**Импорт i18n:**
```typescript
import { i18n } from './i18n';
```

**Заменить функцию `translateState`:**
```typescript
export function translateState(state: string): string {
  // Используем локализацию, но оставляем fallback на английский
  return i18n.t(`table.states.${state}`, { defaultValue: state });
}
```
*Примечание: в i18n.t нужно добавить поддержку defaultValue. Временно можно просто возвращать `i18n.t('table.states.' + state)` и если ключа нет, то вернётся сам `table.states.xxx`, но лучше доработать i18n. Для простоты пока оставим как есть, но потом доработаем.*

**Заменить функцию `formatError`:**
```typescript
export function formatError(errorMessage: string): string {
  const MAX_ERROR_MESSAGE_LENGTH = 500;
  const truncatedMessage = errorMessage.length > MAX_ERROR_MESSAGE_LENGTH
    ? errorMessage.substring(0, MAX_ERROR_MESSAGE_LENGTH) + "... [truncated]"
    : errorMessage;
  return i18n.t('table.error_prefix') + truncatedMessage;
}
```

**Заменить функцию `createDefaultTableBlock` (она не локализуется, т.к. это SQL-код, но можно оставить как есть).**

**Функция `formatFieldValue` – не требует изменений, так как значения полей (числа, даты) уже форматируются отдельно, а перевод состояния уже сделан.**

#### 3. Модифицировать `src/utils/fsrs-table-generator.ts`

**Импорт i18n:**
```typescript
import { i18n } from './i18n';
```

**В `generateTableHTML` заменить информационное сообщение:**
```typescript
if (totalCards > limit) {
  const hiddenCount = totalCards - limit;
  html += `<div class="fsrs-table-info">
    <small>${i18n.t('table.showing_limit', { shown: limit, total: totalCards, hidden: hiddenCount })}</small>
  </div>`;
}
```

**Заголовки колонок:** в текущей реализации заголовки берутся из `params.columns[].title`. При парсинге SQL-блока заголовки задаются пользователем через `as "Заголовок"`. Если пользователь не указал алиас, то используется имя поля. Нам не нужно переводить пользовательские алиасы, но нужно перевести значения по умолчанию (когда алиас не задан). Это нужно сделать на этапе парсинга, но проще – при генерации HTML, если заголовок совпадает с именем поля, то заменить на локализованный. Но это может нарушить пользовательские алиасы. Более правильный подход: при создании `TableParams` в парсере, если алиас не задан, подставить локализованное название. Однако парсер находится в Rust, и это сложнее. Временно оставим как есть – заголовки колонок будут на том языке, который пользователь указал в алиасе, или на английском (имя поля). В будущем можно доработать.

Для целей этого задания – не меняем логику заголовков, только информационные сообщения и состояния.

**В методе `renderEmptyState()`:**
```typescript
private renderEmptyState() {
  if (!this.isFirstLoad) {
    this.container.style.opacity = "0.7";
    this.container.style.transition = "opacity 0.3s ease";
  }
  this.container.empty();
  const emptyDiv = this.container.createDiv({ cls: "fsrs-table-empty" });
  emptyDiv.createEl("small", { text: i18n.t('table.no_cards') });
  if (!this.isFirstLoad) {
    this.container.style.opacity = "1";
  }
}
```

**В методе `renderErrorState()` – сообщение об ошибке уже форматируется через `formatError`, который мы перевели, поэтому дополнительных изменений не требуется.**
