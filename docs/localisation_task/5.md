## Задание для агента №5: Локализация справки `fsrs-help-modal`

### Цель
Перевести всё содержимое модального окна справки по синтаксису `fsrs-table` (команда "Показать справку по синтаксису fsrs-table"). Справка содержит большое количество текста на русском языке (markdown-разметка с примерами, описания полей, параметров). Необходимо вынести этот текст в файлы локализации (английский и русский), чтобы справка отображалась на выбранном языке.

### Контекст
Файл: `src/ui/fsrs-help-modal.ts`. В нём функция `generateHelpText()` возвращает строку с markdown-содержимым справки на русском языке. Вся справка жёстко зашита в коде.

### Задачи

#### 2. Модифицировать `fsrs-help-modal.ts`

**Импорт i18n:**
```typescript
import { i18n } from '../utils/i18n';
```

**Удалить функцию `generateHelpText()`** (она больше не нужна).

**Переписать метод `onOpen()`:**

Вместо вызова `generateHelpText()` и `MarkdownRenderer.render` использовать динамическую сборку содержимого из локализации. Однако проще всего оставить `MarkdownRenderer.render`, но передавать ему строку, собранную из локализованных фрагментов. Для этого создать новую функцию `getLocalizedHelpText()` внутри класса или вне его:

```typescript
private getLocalizedHelpText(): string {
  const help = i18n.t('help');
  // help - это объект, но i18n.t возвращает строку, если путь ведёт к строке.
  // Для составных частей нужно вызывать i18n.t для каждого раздела отдельно.
  // Лучше собрать markdown вручную, используя шаблонные строки и вызовы i18n.t.

  return `
# ${i18n.t('help.title')}

${i18n.t('help.intro')}

## ${i18n.t('help.examples.heading')}

${i18n.t('help.examples.actual')}

${i18n.t('help.examples.sort')}

${i18n.t('help.examples.full')}

## ${i18n.t('help.available_fields.heading')}

${Object.entries(i18n.t('help.available_fields.descriptions')).map(([field, desc]) => `1. **\`${field}\`** — ${desc}`).join('\n')}

## ${i18n.t('help.parameters.heading')}

- ${i18n.t('help.parameters.select')}
- ${i18n.t('help.parameters.select_star')}
- ${i18n.t('help.parameters.order_by')}
- ${i18n.t('help.parameters.limit')}

### ${i18n.t('help.parameters.select_format')}
${i18n.t('help.parameters.select_format')}

### ${i18n.t('help.parameters.order_by_format')}
${i18n.t('help.parameters.order_by_format')}

${i18n.t('help.parameters.notes')}

## ${i18n.t('help.general.heading')}

${i18n.t('help.general.description')}

${i18n.t('help.general.update_note')}

## ${i18n.t('help.general.notes.heading')}

- ${i18n.t('help.general.notes.cache')}
- ${i18n.t('help.general.notes.click_file')}
- ${i18n.t('help.general.notes.click_row')}
- ${i18n.t('help.general.notes.default_sort')}
- ${i18n.t('help.general.notes.custom_sort')}
- ${i18n.t('help.general.notes.rounding')}
`;
}
```

**Затем в `onOpen()`:**
```typescript
const helpMarkdown = this.getLocalizedHelpText();
await MarkdownRenderer.render(this.app, helpMarkdown, contentContainer, "", null as any);
```

**Убедиться, что заголовок и информационная строка также локализованы:**

#### 3. Удалить старую функцию `generateHelpText()` и все связанные с ней русские строки из кода.

#### 4. Убедиться, что примеры в справке корректно отображаются (особенно обратные кавычки). В JSON строки нужно экранировать обратные кавычки и кавычки. Например, для markdown-блока с тройными обратными кавычками в JSON писать:
```json
"actual": "### Actual overdue cards (most urgent)\n```fsrs-table\nSELECT file as \"File\", reps as \"Reps\", overdue as \"Overdue\", retrievability as \"Retrievability\"\nLIMIT 20\n```"
```
Обратите внимание: внутри строки JSON кавычки экранируются `\"`, а обратные кавычки не требуют экранирования. Перевод строки – `\n`.

#### 5. Проверить, что в `ru.json` все переводы соответствуют оригинальному тексту из `generateHelpText()`, включая примеры с русскими алиасами (например, `"Файл"`, `"Повт."` и т.д.). Сохранить все специальные символы (стрелки, эмодзи).

### Критерии выполнения
- Вся справка вынесена в файлы локализации, в коде не осталось русских строк (кроме комментариев).
- При открытии справки (команда "Показать справку по синтаксису fsrs-table") на английском языке отображается английская версия, на русском – русская.
- Markdown-разметка корректно рендерится Obsidian (примеры кода, списки, заголовки).
- Все описания полей, параметров и примечания переведены.
- Размер окна, стили и поведение не изменились.

### Проверка
- Открыть справку при локали 'en' – проверить, что текст на английском, примеры используют английские алиасы.
- Переключить локаль на 'ru' (временно через `i18n.setLocale('ru')`) – справка должна стать русской, примеры с русскими алиасами.
- Убедиться, что ссылки, примеры кода и форматирование работают.
- Проверить, что нет ошибок в консоли.
