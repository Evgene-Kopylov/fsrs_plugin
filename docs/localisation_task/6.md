## Задание для агента №6: Локализация настроек плагина и добавление выбора языка

### Цель
Перевести все текстовые элементы на вкладке настроек плагина (FSRS Setting Tab) с использованием системы локализации. Добавить выпадающий список (dropdown) для выбора языка интерфейса (английский/русский) и реализовать динамическое обновление интерфейса настроек при смене языка без перезагрузки плагина (или с перезагрузкой, но лучше без неё).

### Контекст
Файлы настроек:
- `src/settings/settings-core.ts` – основной класс вкладки, вызывает функции рендеринга групп.
- `src/settings/setting-groups/fsrs-parameters.ts` – параметры алгоритма.
- `src/settings/setting-groups/card-defaults.ts` – настройки по умолчанию для новых карточек.
- `src/settings/setting-groups/display-settings.ts` – настройки отображения (кнопка, иконка статус-бара).
- `src/settings/setting-groups/early-review-settings.ts` – настройки досрочного повторения.
- `src/settings/setting-groups/filtering-settings.ts` – паттерны игнорирования файлов.

Все эти файлы содержат жёстко зашитые русские строки в вызовах `.setName()`, `.setDesc()`, `.setPlaceholder()` и т.д.

Также нужно добавить новый раздел настроек для выбора языка (например, после параметров алгоритма).

#### 3. Модифицировать все файлы групп настроек для использования локализации

**Общий паттерн для каждого файла:**

- Импортировать `i18n` из `'../utils/i18n'` (путь может отличаться, используйте относительный).
- Заменить все строки в `setName()`, `setDesc()`, `setPlaceholder()` на вызовы `i18n.t('settings.xxx.yyy')`.

**Пример для `fsrs-parameters.ts`:**

```typescript
import { Setting } from "obsidian";
import type MyPlugin from "../../main";
import { i18n } from "../../utils/i18n";

export function renderFsrsParameters(containerEl: HTMLElement, plugin: MyPlugin): void {
  new Setting(containerEl).setName(i18n.t('settings.fsrs_algorithm.heading')).setHeading();

  new Setting(containerEl)
    .setName(i18n.t('settings.fsrs_algorithm.request_retention.name'))
    .setDesc(i18n.t('settings.fsrs_algorithm.request_retention.desc'))
    .addSlider((slider) => slider
      .setLimits(0.5, 1.0, 0.001)
      .setValue(plugin.settings.parameters.request_retention)
      .setDynamicTooltip()
      .onChange(async (value) => { ... })
    );

  // аналогично для maximum_interval и enable_fuzz
}
```

Аналогично для остальных групп:
- `card-defaults.ts`
- `display-settings.ts`
- `early-review-settings.ts`
- `filtering-settings.ts`

#### 4. Добавить новый раздел выбора языка в `settings-core.ts`

В методе `display()` после рендеринга параметров FSRS (или в любом удобном месте) добавить новый блок:

```typescript
import { i18n } from "../utils/i18n";

// ... внутри display() после renderFsrsParameters

new Setting(containerEl).setName(i18n.t('settings.language.heading')).setHeading();

new Setting(containerEl)
  .setName(i18n.t('settings.language.heading'))
  .setDesc(i18n.t('settings.language.desc'))
  .addDropdown(dropdown => {
    dropdown
      .addOption('en', i18n.t('settings.language.en'))
      .addOption('ru', i18n.t('settings.language.ru'))
      .setValue(plugin.settings.language || 'en')
      .onChange(async (value: 'en' | 'ru') => {
        plugin.settings.language = value;
        await plugin.saveSettings();
        i18n.setLocale(value);
        // Перерисовать вкладку настроек для применения нового языка
        this.display();
      });
  });
```

**Важно:** При смене языка через dropdown нужно:
- Сохранить новое значение в настройках.
- Обновить локаль в `i18n`.
- Перерисовать всю вкладку настроек (вызвать `this.display()`), чтобы все тексты обновились.

#### 5. В `main.ts` при загрузке настроек установить локаль

Уже было в предыдущих фазах, но убедиться:

```typescript
async loadSettings() {
  this.settings = Object.assign({}, DEFAULT_SETTINGS, await this.loadData());
  i18n.setLocale(this.settings.language || 'en');
}
```

#### 6. Обновить `i18n.ts` для поддержки динамической смены языка и перерисовки

Убедиться, что в `i18n.ts` есть метод `setLocale`, который обновляет текущий язык и перезагружает переводы. Также может быть полезно добавить событие, чтобы другие компоненты (статус-бар, таблица) могли обновиться, но для настроек достаточно перерисовать вкладку.

### Критерии выполнения
- Все тексты на вкладке настроек заменены на вызовы `i18n.t()`.
- Добавлен выпадающий список выбора языка (английский/русский).
- При смене языка в настройках весь интерфейс настроек переключается на выбранный язык без перезагрузки плагина (перерисовка вкладки).
- Значение языка сохраняется в настройках и восстанавливается при следующем запуске.
- При первом запуске (или если язык не задан) используется английский язык по умолчанию.
- Другие части плагина (статус-бар, таблица, справка, уведомления) также должны реагировать на смену языка – для этого нужно либо вызывать обновление соответствующих компонентов при смене языка (сложно), либо просто перезагрузить плагин (проще, но неудобно). В данном задании достаточно, чтобы настройки перерисовывались. Остальные компоненты обновятся при следующем действии пользователя (открытие модалки, повторение карточки и т.д.), так как они каждый раз читают текущую локаль. Это приемлемо.

### Проверка
- Открыть настройки плагина – все заголовки, описания, плейсхолдеры должны быть на английском.
- Выбрать русский язык в dropdown – вся вкладка должна перерисоваться на русском.
- Закрыть и снова открыть настройки – язык должен сохраниться.
- Проверить, что при смене языка другие части плагина (например, команды, уведомления) используют новый язык (это уже реализовано в предыдущих фазах).

### Результат
Настройки плагина полностью локализованы. Пользователь может выбрать язык интерфейса. Все тексты вынесены в файлы локализации. Плагин готов к использованию с поддержкой двух языков (английский и русский).

### Примечание для агента
- Будьте внимательны с путями импорта `i18n` в каждом файле настроек (относительный путь до `../../utils/i18n`).
- В `filtering-settings.ts` используется переменная `configDir` – её не нужно локализовать.
- Убедитесь, что после добавления поля `language` в `types.ts` и `DEFAULT_SETTINGS` код компилируется.
- При перерисовке вкладки настроек через `this.display()` не теряются текущие значения полей (они считываются заново из `plugin.settings`). Это корректно.
