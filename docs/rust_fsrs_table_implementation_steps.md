# Детальный пошаговый план реализации переноса парсинга fsrs-table на Rust

## Общие принципы
- ✅ **Нет обратной совместимости** - можно менять форматы выходных данных
- ✅ **Полный перенос логики** в Rust (парсинг, фильтрация, сортировка)
- ✅ **TypeScript остаётся для UI** - форматирование, HTML генерация, рендеринг
- ✅ **Даты в формате Obsidian**: ГГГГ-ММ-ДД_чч:мм (сортировка как строки)

## Порядок работ

Выполнять этап -> тесты -> коммит

### 2.2. Синтаксический анализатор (`parsing/parser.rs`)
- [ ] Определить грамматику:
  ```
  query ::= select_clause [order_by_clause] [limit_clause]
  select_clause ::= "SELECT" column_list
  column_list ::= column_def ("," column_def)*
  column_def ::= identifier ["AS" string]
  order_by_clause ::= "ORDER" "BY" identifier ["ASC" | "DESC"]
  limit_clause ::= "LIMIT" number
  ```
- [ ] Реализовать парсер с обработкой ошибок:

## Примечания

1. **Даты формата Obsidian**: Все даты приводятся к формату `ГГГГ-ММ-ДД_чч:мм` для сортировки как строк
2. **Ошибки парсинга**: Функции возвращают `Result` с детальными сообщениями об ошибках
3. **Производительность**: Логирование через `log` крейт вместо `console.timer`
4. **Тестирование**: Использовать существующие блоки `fsrs-table` для интеграционных тестов
