# Использование универсального блока fsrs-table в FSRS плагине для Obsidian

## 📖 Введение

Блок `fsrs-table` — это **универсальный динамически рендерируемый** блок для отображения карточек FSRS. Он поддерживает два режима отображения: `due` (просроченные) и `all` (все карточки). Плагин автоматически рендерит интерактивный интерфейс прямо на месте блока, аналогично тому, как это делает Journals с `calendar-timeline`.

### Основные принципы:

1. **Универсальный блок**: Один блок для всех задач — `due` и `all`
2. **Гибкие параметры**: Настройка режима, количества карточек, колонок
3. **Динамический рендеринг**: Автоматическая загрузка и фильтрация карточек при открытии файла
4. **Интерактивность**: Кликабельные ссылки, автоматическое обновление, цветовое кодирование

## 🚀 Как использовать

### 1. Добавить блок fsrs-table в файл

Добавьте в любое место вашего markdown-файла:

````markdown
```fsrs-table
```
````

По умолчанию блок покажет карточки для повторения (режим `due`).

### 2. SQL-подобный синтаксис

````markdown
```fsrs-table
SELECT file, reps, state, stability, difficulty, overdue, due
FROM due
LIMIT 30
```
````

### 3. Автоматический рендеринг

При открытии файла плагин автоматически:
1. Просканирует всё хранилище на наличие карточек FSRS
2. Отфильтрует карточки в зависимости от выбранного режима
3. Отрендерит интерактивную таблицу на месте блока

### 4. Где использовать блок

Вы можете добавить блок `fsrs-table` в любой markdown-файл:
- **Ежедневные заметки**: Для ежедневного повторения (режим `due`)
- **Статистика и мониторинг**: Для полного обзора всех карточек (режим `all`)
- **Панель управления**: Создайте отдельный файл как панель управления карточками

## ⚙️ Параметры блока

### Основные параметры:

### SQL-подобный синтаксис

Блок `fsrs-table` поддерживает SQL-подобный синтаксис:

```
SELECT field1 as "Alias1", field2, field3
FROM due | all
ORDER BY field [ASC | DESC]
LIMIT N
```

#### Основные правила:
- **FROM** – обязателен, определяет режим: `due` или `all`
- **SELECT** – опционален. Список полей через запятую. Поддерживается алиас `field as "Alias"`. Если SELECT отсутствует, используются поля по умолчанию: `file, reps, overdue`
- **ORDER BY** – опционален. Указывает поле и направление сортировки (ASC или DESC). Если отсутствует, применяется стандартная сортировка FSRS
- **LIMIT** – опционален. Целое число, ограничивающее количество строк. Если отсутствует, используется значение из настроек плагина
- Ключевые слова могут быть в любом порядке. Регистр не важен. Поля и алиасы сохраняют регистр

### Режимы работы (FROM clause):

- **`FROM due`** — Карточки, готовые к повторению (просроченные)
  - Показывает только карточки, у которых дата повторения наступила
  - Сортирует по приоритету (самые старые первыми), если не указан ORDER BY

- **`FROM all`** — Все карточки FSRS
  - Показывает все карточки независимо от статуса
  - Сортирует по дате следующего повторения, если не указан ORDER BY

### Доступные поля (SELECT clause):

| Поле | Описание | Пример значения |
|---------|----------|-----------------|
| `file` | Имя файла (кликабельная ссылка) | `cell-structure.md` |
| `reps` | Количество повторений | `5` |
| `overdue` | Время просрочки | `2 дня 3 часа` |
| `stability` | Стабильность памяти (дни) | `12.5` |
| `difficulty` | Сложность карточки (0.0–1.0) | `0.45` |
| `retrievability` | Извлекаемость (в процентах) | `65%` |
| `due` | Дата следующего повторения | `2024-01-20, 14:30` |
| `state` | Состояние карточки | `Review` |
| `elapsed` | Прошедших дней | `3` |
| `scheduled` | Запланированных дней | `7` |

## 🎯 Примеры использования

### 1. Для ежедневного повторения (режим `due`):

````markdown
```fsrs-table
mode: due
limit: 50
columns: file,reps,state,overdue
```
````

*Что показывает:*
- До 50 просроченных карточек
- Только основные колонки для быстрого обзора
- Сортировка по приоритету (самые забытые первыми)

### 2. Для полного обзора (режим `all`):

````markdown
```fsrs-table
mode: all
limit: 200
columns: file,reps,state,due
```
````

*Что показывает:*
- Все карточки FSRS в хранилище (до 200)
- Основная информация о состоянии и расписании
- Полная картина вашего прогресса обучения

### 3. Минималистичная версия:

````markdown
```fsrs-table
mode: due
limit: 10
columns: file,overdue
```
````

## 🧭 Навигация и интерактивность

### Кликабельные элементы:

1. **Ссылки на файлы**: Имя файла является кликабельной ссылкой для перехода к карточке
2. **Целая строка**: Клик на любой части строки открывает файл с карточкой
3. **Автоматическое обновление**: Список обновляется при каждом открытии файла

### Сортировка карточек:

- **Режим `due`**: Сортировка по извлекаемости (самые забытые первыми)
- **Режим `all`**: Сортировка по дате следующего повторения

### Лимит отображения:

По умолчанию отображается **первые 30 карточек**:
- Внизу указывается количество скрытых карточек
- Например: `Показано: 30 из 45 карточек (15 скрыто)`
- Лимит можно изменить параметром `limit`

## 🔧 Техническая реализация

### Архитектура:

1. **`registerMarkdownCodeBlockProcessor`**:
   - Регистрирует обработчик для блоков `fsrs-table`
   - Автоматически вызывается при рендеринге markdown
   - Передает управление классу `FsrsTableRenderer`

2. **Класс `FsrsTableRenderer`**:
   - Наследуется от `MarkdownRenderChild`
   - Парсит параметры блока (mode, limit, columns)
   - Загружает карточки через `plugin.getCardsForReview()`
   - Фильтрует и сортирует карточки в зависимости от режима
   - Рендерит HTML таблицу с обработчиками событий

3. **Фильтрация и сортировка**:
   - `due`: Использует `filterCardsForReview()` для фильтрации просроченных карточек
   - `all`: Показывает все карточки без фильтрации

### Процесс рендеринга:

```typescript
1. Пользователь открывает файл с блоком ```fsrs-table```
2. Obsidian вызывает registerMarkdownCodeBlockProcessor
3. Создается экземпляр FsrsTableRenderer с параметрами
4. Рендерер показывает индикатор загрузки
5. Асинхронно загружаются все карточки FSRS
6. Карточки фильтруются и сортируются по выбранному режиму
7. Динамически генерируется HTML таблица
8. Добавляются обработчики событий для ссылок и строк
```

1. **При открытии файла**: Список всегда актуален

## 🚨 Решение проблем

### Проблема: "Блок fsrs-table пустой"
**Решение**: 
1. Проверьте, что у вас есть карточки с полями FSRS в frontmatter
2. Убедитесь, что в выбранном режиме есть карточки (например, в режиме `due` нужны просроченные карточки)
3. Выполните команду "Найти карточки для повторения" для принудительного обновления
4. Проверьте параметры `limit` (возможно, установлен слишком маленький лимит)

### Проблема: "Неправильные колонки или их порядок"
**Решение**:
1. Проверьте синтаксис параметра `columns` (через запятую, без пробелов)
2. Убедитесь, что используете правильные названия колонок (см. раздел "Доступные колонки")
3. Пример правильного синтаксиса: `columns: file,reps,state,due`

### Проблема: "Клики по ссылкам не работают"
**Решение**:
1. Перезагрузите Obsidian
2. Проверьте, включен ли плагин в настройках
3. Убедитесь, что файлы с карточками существуют по указанным путям

### Проблема: "Медленная загрузка"
**Решение**:
1. Уменьшите параметр `limit` (например, до 20)
2. Используйте меньше колонок в параметре `columns`
3. Для больших хранилищ используйте блок только в нужных файлах
4. В будущих версиях планируется кэширование результатов

## 💡 Советы и рекомендации

### Оптимальное использование:

1. **Ежедневный ритуал**: Добавьте блок в ваш daily note template с режимом `due`
3. **Мониторинг прогресса**: Добавьте блок с режимом `all` в файл статистики
4. **Разные представления**: Создайте несколько файлов с разными настройками блока

### Интеграция с другими плагинами:

- **Dataview**: Экспортируйте данные из блока для сложного анализа
- **Tasks**: Создавайте задачи на основе карточек для повторения
- **Calendar**: Планируйте повторения в календаре на основе дат из блока
- **Templater**: Автоматически добавляйте блок в новые файлы по шаблону

### Производительность:

1. **Размер хранилища**: Для хранилищ > 1000 файлов первое сканирование может занять 2-3 секунды
2. **Параметр limit**: Установите разумный лимит для быстрой загрузки
3. **Колонки**: Чем меньше колонок, тем быстрее рендеринг
4. **WASM инициализация**: Загрузка WASM модуля происходит один раз при старте плагина
