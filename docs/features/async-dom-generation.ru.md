# Асинхронная генерация таблицы fsrs-table

## Проблема

При открытии заметки с блоком `fsrs-table` в большом хранилище (10 000 карточек) Obsidian зависал на десятки секунд. Ни скроллить, ни переключаться между заметками в это время было нельзя.

Две причины:

1. **Повторный расчёт** — при первом открытии таблица заново вычисляла состояния всех карточек, хотя они уже были посчитаны.
2. **Нет ограничения** — если в запросе не указан лимит, таблица пыталась отобразить все 10 000 строк.

## Решение

Сделали две вещи:

1. **Используем готовые данные** — при первом открытии таблица берёт уже посчитанные состояния карточек, не пересчитывая их заново. Время открытия упало с ~23 секунд до 1–2 секунд. → `src/ui/fsrs-table-renderer.ts#L182-192`

2. **Лимит по умолчанию** — даже если в запросе не указано ограничение, таблица показывает не больше 200 строк. Снизу пишет: «Показано 200 из 10000». → `src/utils/fsrs-table-generator.ts#L40-42`

## Что изменилось для пользователя

- **Раньше**: открываешь заметку → Obsidian зависает на 10–20 секунд → резко появляется вся таблица.
- **Теперь**: открываешь заметку → таблица появляется за 1–2 секунды, интерфейс остаётся отзывчивым.
