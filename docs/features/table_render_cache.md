## Статический кэш рендерера таблиц fsrs-table

### Проблема

При каждом переключении между заметками Obsidian пересоздаёт экземпляр `FsrsTableRenderer`. Новый экземпляр не имеет кэша — `cachedCardsWithState = null`. В результате каждый раз:

1. Загружаются **все** 20 000+ карточек из `IncrementalCache`
2. Все 20 000 передаются в WASM для фильтрации и сортировки
3. WASM парсит JSON, фильтрует по WHERE, возвращает результат

Даже с WHERE-условием, выбирающим 177 карточек, WASM обрабатывает все 20 000.  
Время загрузки таблицы при возвращении — **~7 секунд**.

### Решение

Добавлен **статический кэш отфильтрованных карточек** на уровне класса `FsrsTableRenderer`:

```typescript
private static rendererCache = new Map<string, RendererCacheEntry>();

interface RendererCacheEntry {
    cardsWithState: CardWithState[];
    totalCount: number;
    params: TableParams | null;
    sourceText: string;
}
```

**Как работает:**

- **Ключ кэша** — `sourcePath` (путь к md-файлу с блоком fsrs-table)
- **Проверка валидности** — сравнивается `sourceText` (SQL-запрос). Если SQL изменился — кэш считается устаревшим
- **Жизненный цикл:**
  1. При первом рендере результат сохраняется в статический кэш
  2. При возвращении на заметку новый экземпляр рендерера проверяет статический кэш, находит сохранённые карточки и восстанавливает `cachedCardsWithState`, `params`, `cachedTotalCount`
  3. При `forceRefresh` (вызов после ревью карточек) — кэш очищается, таблица перезагружается с новыми данными

### Преимущества

- **Мгновенное возвращение** — таблица появляется сразу, без загрузки 20 000 карточек
- **Корректность** — кэш инвалидируется при изменении SQL или после ревью
- **Прозрачность** — не требует изменений в архитектуре плагина, использует существующий `IncrementalCache`

### Связанные оптимизации

- Полное сканирование разбито на **чанки по 100 файлов** с `setTimeout` между ними — UI не блокируется при первой загрузке
- `refresh(forceRefresh)` — без `forceRefresh` не сбрасывает кэш, только перерисовывает

### Проверка

- `npm run build` — без ошибок. ✅
- При первом открытии таблицы — загрузка 20 000 карточек, фильтрация (7 с).
- При возвращении на заметку — мгновенно (~0 с).
- После ревью — перезагрузка таблицы с новыми данными.
