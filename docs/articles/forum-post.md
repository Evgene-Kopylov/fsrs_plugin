# FSRS для Obsidian — современный алгоритм интервального повторения с SQL-запросами к карточкам

Всем привет! 👋

Хочу рассказать о плагине, который делал последние несколько месяцев — **FSRS для Obsidian**. Если вы пользуетесь интервальным повторением в Obsidian, возможно, вам будет интересно.

## Что не так с тем, что есть?

На данный момент в Obsidian есть три популярных плагина для повторений, и все они используют SM-2 — алгоритм, которому почти 40 лет. Он был хорош для своего времени, но:

- Все карточки получают одинаковый интервал, независимо от их сложности
- Если вы пропустили несколько дней — прогресс сбрасывается
- Нет понятия «извлекаемости» — вероятности, что вы вспомните карточку прямо сейчас

**FSRS (Free Spaced Repetition Scheduler)** — это современная замена. Он использует машинное обучение (21 параметр, подобранный на миллионах реальных повторений) и требует на 20–30% меньше повторений для того же уровня запоминания. Если вам интересна теория — вот [ABC of FSRS](https://github.com/open-spaced-repetition/awesome-fsrs/wiki/ABC-of-FSRS).

## Что умеет плагин

Плагин превращает обычные заметки Obsidian в карточки для запоминания. Достаточно добавить в frontmatter поле `reviews: []` — и заметка становится карточкой FSRS.

**Вот как это выглядит в работе:**

> *[тут будет гифка: открываешь заметку с fsrs-table, видишь таблицу карточек, наводишь на строку — всплывает содержимое, жмёшь оценку — таблица обновляется]*

### Ключевые фичи

**🔍 SQL-подобные запросы к карточкам**
Это главная «фишка», которой нет в других плагинах. Вы пишете прямо в заметке:

````markdown
```fsrs-table
SELECT file as "Карточка", difficulty as "D",
       stability as "S", retrievability as "R",
       due as "Следующее"
WHERE retrievability < 0.9
ORDER BY due ASC
LIMIT 20
```
````

И получаете живую таблицу, которая фильтрует, сортирует и обновляется сама после каждого повторения. Можно вывести «только карточки, которые пора повторять» (`retrievability < 0.9`), «только новые» (`state = "new"`), «просроченные» — любой SQL-подобный запрос.

**🖱️ Повторение без перехода к заметке**
Наводите курсор на имя файла в таблице — появляется всплывающее окно с содержимым карточки и кнопками Again/Hard/Good/Easy. Можно пролистать десяток карточек за пару минут, не открывая ни одной заметки.

**⚡ Живые анимации**
Таблицы не просто обновляются — строки с анимацией «переезжают» на новые позиции, новые карточки плавно появляются. Это сделано не для красоты, а чтобы глаз отслеживал изменения, особенно когда сортируешь по извлекаемости и видишь, что самая «горящая» карточка уезжает наверх.

**📱 Работает на мобильных**
Плагин использует Rust/WASM для вычислений — никаких внешних сервисов, всё локально. Работает и на iOS, и на Android.

**🌐 Локализация**
Русский, английский, китайский — язык автоматически подхватывается из настроек Obsidian.

## Быстрый старт

1. Установите плагин через BRAT: `https://github.com/Evgene-Kopylov/fsrs_plugin`
2. Откройте заметку → команда `＋ FSRS: Добавить поля FSRS в шапку файла` (добавит `reviews: []` в frontmatter)
3. Вставьте блок ` ```fsrs-review-button``` ` — появится кнопка для оценки карточки
4. Создайте заметку с ` ```fsrs-table``` ` — будет таблица со всеми карточками

Подробный гайд с картинками — [тут](https://gitlab.com/Evgene-Kopylov/FSRS-plugin).

## Техническая часть (коротко)

Плагин написан на TypeScript + Rust. Вся логика FSRS и SQL-подобный парсер живут в Rust и компилируются в WebAssembly. TypeScript занимается только Obsidian API, UI и файловой системой. Получилось быстро и надёжно — 174 теста в Rust, 57 в TypeScript, все без моков.

## Текущий статус

Версия **0.9.5**, предрелизная стадия. Базовый цикл (создание карточки → повторение → таблица) работает стабильно. Интерфейсы ещё могут немного меняться, но я стараюсь не ломать совместимость без крайней необходимости.
