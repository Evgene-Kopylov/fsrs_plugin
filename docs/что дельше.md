# Варинты дальнейших дейсвий

Исходя из анализа проекта, у вас уже реализована значительная часть функционала FSRS плагина для Obsidian. Основные компоненты работают:

- ✅ **WASM интеграция** с Rust библиотекой rs-fsrs
- ✅ **Три основные команды**: добавление полей FSRS, поиск карточек для повторения, повторение карточки с модальным окном оценки
- ✅ **Настройки алгоритма FSRS** с параметрами request_retention, maximum_interval, enable_fuzz
- ✅ **Блок `fsrs-now`** для отображения карточек в markdown
- ✅ **Парсинг нового формата карточек** с массивом reviews

## 🎯 Что можно сделать дальше (выберите направление)

### 1. **Улучшение UI оценки карточки** (высокий приоритет)
Добавить в модальное окно оценки отображение следующих дат повторения для каждой оценки (Again/Hard/Good/Easy). Сейчас пользователь не видит, когда будет следующее повторение при выборе каждой оценки.

### 2. **Панель View для управления карточками** (высокий приоритет)
Создать боковую панель Obsidian (ItemView) для:
- Отображения списка карточек для повторения
- Сортировки по дате, извлекаемости (retrievability), сложности
- Фильтрации по папкам, тегам, состоянию
- Быстрого перехода к карточке

### 3. **Интеграция с Obsidian UI** (средний приоритет)
- Добавить ribbon icon для быстрого открытия панели или выполнения команд
- Реализовать автоматическое обновление карточек при изменении файлов
- Добавить уведомления о количестве карточек для повторения

### 4. **Статистика и аналитика** (средний приоритет)
- График кривой забывания для карточки
- Статистика по повторениям (успешность, количество)
- Прогресс обучения (карточки по состояниям: New, Learning, Review, Relearning)

### 5. **Технический долг и тесты** (из NOTES.md)
- Исправить предупреждения Rust (`cargo clippy --fix`)
- Настроить строгий парсинг фронтматтера (избегать ложных срабатываний)
- Добавить модульные тесты для интеграции с WASM
- Рефакторинг больших файлов (особенно `src/utils/fsrs/fsrs-wasm.ts`)

### 6. **Подготовка к релизу**
- Обновить `manifest.json` (id, name, description, version)
- Создать пользовательскую документацию в README
- Настроить автоматическую сборку и публикацию

### 7. **Оптимизация производительности**
- Динамическая загрузка WASM вместо base64 встраивания
- Кэширование результатов поиска карточек
- Фоновая обработка для больших хранилищ

## 💡 Рекомендация
Судя по плану и текущему состоянию, наиболее логичные следующие шаги:

1. **Сначала исправить технический долг** — предупреждения Rust и улучшение парсинга фронтматтера (задачи из NOTES.md).
2. **Затем добавить View панель** — это значительно улучшит UX и будет соответствовать пункту "Улучшение поиска карточек" из плана.
3. **Дополнить модальное окно** отображением следующих дат повторения.
