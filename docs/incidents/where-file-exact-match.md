# WHERE file = "..." не находит файлы

## Описание

`SELECT ... WHERE file = "имя_файла"` не возвращает строк, даже когда файл с таким именем существует и есть в кэше.

## Причина

Поле `file` в `CardWithComputedFields` заполняется из `card.file_path` — полного пути в хранилище (например `папка/2026-04-19_Запись.md`).

Фильтр `WHERE file = "2026-04-19_Запись_17-27-25"` делает **точное сравнение** с полным путём — `.md` расширение и путь к папке не совпадают → 0 строк.

## Воспроизведение

```sql
SELECT file
WHERE file = "имя-файла-без-пути-и-расширения"
```

## Ожидаемое поведение

`file = "..."` должно находить файл по имени, без необходимости указывать `.md` и путь к папке. Либо:

- совпадение по имени файла без расширения
- поддержка `LIKE`/частичного совпадения
