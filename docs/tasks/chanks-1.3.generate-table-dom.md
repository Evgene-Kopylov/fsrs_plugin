# Задача 1.3: Адаптация generateTableDOM для работы с неполными данными

**Файл:** `src/utils/fsrs-table-generator.ts`

## Описание

Сейчас `generateTableDOM` уже принимает `cardsWithState` и отображает первые `limit` строк — ничего менять не нужно, функция уже работает с любым подмножеством данных. Однако нужно добавить пользовательскую обратную связь о том, что сканирование ещё не завершено.

## Задачи

- [ ] Добавить опциональное сообщение внизу таблицы: *"Загружено X карточек, сканирование продолжается…"*
  - Сообщение должно появляться только если сканирование ещё не завершено
  - Реализовать через колбэк или флаг, передаваемый в `generateTableDOM`
- [ ] Убедиться, что при завершении сканирования сообщение исчезает
- [ ] Проверить, что при `limit` сообщение не конфликтует с пагинацией/усечением строк
