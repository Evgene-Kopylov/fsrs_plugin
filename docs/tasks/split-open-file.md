# Открытие файла из таблицы в соседней вкладке

При клике на файл в таблице `fsrs-table` — открывать в соседней
markdown-вкладке, если она есть (сплит или отдельное окно).
Если соседней вкладки нет — стандартное поведение Obsidian.

## Что пробовали

| #   | Подход                                                                                                     | Результат                                                                                                                       |
| --- | ---------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------- |
| 1   | `getLeaf("split")` всегда                                                                                  | сплит создаётся, даже если не было                                                                                              |
| 2   | `getLeavesOfType("markdown").length > 1` → `leaves[last]`                                                  | работает, но только в одну сторону; если таблица в доп. окне — открывает в нём же                                               |
| 3   | `e.target` + `contains` → `find()`                                                                         | работает в обе стороны, но по порядку создания, не по последнему использованию                                                  |
| 4   | `mousedown` + `getMostRecentLeaf()`                                                                        | `mousedown` обрабатывает, но `click` всё равно срабатывает — двойное открытие                                                   |
| 5   | `mousedown` запоминает лиф, `click` открывает                                                              | `getMostRecentLeaf()` на `mousedown` возвращает таблицу, а не предыдущий лиф                                                    |
| 6   | вариант 3 + `offsetParent !== null` — только видимые лифы                                                  | скрытый лиф всё равно выбирался                                                                                                 |
| 7   | `mousedown` capture + трекинг + приоритет: empty → markdown → любой                                        | empty-таб первый, затем last used markdown, затем iterateAllLeaves. Только `.mod-root`.                                         |
| 8   | трекинг последних 2 лифов + `findLeafContaining` + приоритет: empty → отслеженный → любой markdown → любой | `tableClickInProgress` убран (залипал). `currentLeaf` находится через `iterateAllLeaves` + `contains`. Без `mousedown` capture. |

## Текущее состояние

Работает вариант 8: отслеживание двух последних markdown-лифов через
`active-leaf-change`, поиск текущего лифа через `findLeafContaining()`.
Приоритет целевого лифа: empty → отслеженный markdown → любой markdown → любой не-сайдбар.
Если целевой лиф не найден — стандартное поведение Obsidian (без `preventDefault`).
