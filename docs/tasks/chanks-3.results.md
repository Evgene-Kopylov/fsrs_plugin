# Задача 3: Оформление результатов

## Формат коммитов

Каждый логический шаг — отдельный коммит с подробным сообщением на русском языке:

| Шаг | Сообщение коммита |
|-----|------------------|
| 0.1 | `refactor: переименован DEFAULT_TABLE_DISPLAY_LIMIT и вынесен в constants` |
| 0.2 | `refactor: разбиение [имя файла] на модули` |
| 1.1 | `feat: добавлен метод startProgressiveScan в IncrementalCache` |
| 1.2 | `feat: прогрессивный рендеринг таблицы через чанки` |
| 1.3 | `feat: сообщение о прогрессе сканирования в таблицу` |
| 1.4 | `feat: досрочная остановка сканирования по limitHint` |

## Критерий готовности задачи

Прогрессивное сканирование работает и время первого отображения таблицы сократилось:

- **Было:** ~10 секунд до появления таблицы (полное сканирование)
- **Стало:** <2 секунды до появления первых строк на хранилище с 10k файлов

Таблица появляется с первыми данными почти мгновенно, затем дополняется в фоне.

## Порядок выполнения

1. Выполнить подготовительные рефакторинги (0.1, 0.2) — отдельные коммиты
2. Реализовать прогрессивное сканирование (1.1–1.4) — последовательно, каждый шаг отдельным коммитом
3. Провести ручное тестирование (задача 2)
4. После принятия пользователем — отметить все подзадачи выполненными в `index.md`
