## Ситуация

При первом рендере fsrs-table в хранилище с 10 000 карточек интерфейс Obsidian блокируется на несколько секунд.

## Причина

Если в SQL-блоке не указан `LIMIT`, Rust-функция получает `params.limit === 0` и возвращает **все** 10 000 карточек. `generateTableDOM` в синхронном цикле создаёт DOM-элементы для каждой — 10 000 `<tr>` и десятки тысяч `<td>`. UI блокируется на всё время создания.

## План

- [ ] **Не создавать DOM для карточек сверх лимита**
  - Где: `src/utils/fsrs-table-generator.ts`, функция `generateTableDOM`.
  - Что: в начале функции обрезать `cardsToShow` до `params.limit`, если `limit > 0`. Если `limit === 0` — применить лимит по умолчанию (200).
  - ВАЖНО: `totalCount` при этом не трогать — он нужен для сообщения «Показано X из Y».
  - Зачем: физически не создавать тысячи лишних DOM-узлов.

- [ ] **Проверить на хранилище с 10 000 карточек**
  - Собрать бандл (`npm run build`).
  - Открыть заметку с fsrs-table без `LIMIT`.
  - Убедиться, что UI не блокируется, а создаётся только 200 `<tr>`.
  - Проверить сообщение «Показано 200 из 10000».
