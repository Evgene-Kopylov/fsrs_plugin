## Ситуация

При первом рендере fsrs-table в хранилище с 10 000 карточек интерфейс Obsidian блокируется на несколько секунд.

## Причина

Одна из проблем: **WASM не применяет лимит при `LIMIT 0`** — если в SQL-блоке не указан `LIMIT`, или указан `LIMIT 0`, Rust-функция `filter_and_sort_cards` возвращает **все** карточки без ограничения. Все они попадают в синхронную генерацию DOM и блокируют UI.

## План

- [ ] **1. Применить разумный лимит по умолчанию при `limit === 0`**
  - Где: `src/utils/fsrs-table-generator.ts`, на входе в `generateTableDOM`.
  - Что: если `params.limit === 0`, установить значение по умолчанию — 200 строк.
  - Зачем: чтобы WASM не возвращал все 10 000 карточек в DOM.

- [ ] **2. Проверить загрузку на хранилище с 10 000 карточек**
  - Собрать бандл (`npm run build`).
  - Открыть заметку с fsrs-table.
  - Замерить время загрузки.
  - Убедиться, что сообщение «Показано X из Y» корректно.
