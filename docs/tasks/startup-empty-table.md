# Пустая таблица при запуске Obsidian

**Статус:** в работе

## Проблема

При запуске Obsidian таблицы `fsrs-table`, созданные в предыдущих сессиях, показывают «Нет карточек FSRS», хотя в хранилище есть карточки с FSRS-полями.

При этом новосозданные таблицы (добавленные вручную после запуска) работают нормально — находят и отображают карточки.

## Причина

`performCacheScan()` вызывается в `onload()` плагина. Но на момент `onload()` Obsidian ещё не закончил индексацию файлов хранилища — `this.app.vault.getMarkdownFiles()` возвращает **пустой массив**. 

Подтверждение из логов:

```
plugin:fsrs:87 ✅ Найдено карточек FSRS: 0
plugin:fsrs:87 ⏱️ Сканирование всего хранилища: 0.00 с
```

Сканирование за 0.00 с означает, что не было найдено ни одного markdown-файла. При этом Dataview проиндексировал 3009 файлов за 1.65 с уже после нашего сканирования.

**Почему новые таблицы работают?** К моменту, когда пользователь создаёт новую таблицу, Obsidian уже проиндексировал все файлы. Карточки добавляются в кэш через события `modify` (при открытии/чтении файлов). Поэтому запрос `cache.query()` возвращает данные.

## Исправление

1. Перенесён запуск `performCacheScan()` из `onload()` в `this.app.workspace.onLayoutReady()`. `onLayoutReady` гарантирует, что рабочий стол Obsidian полностью проинициализирован, включая индексацию файлов.

2. Добавлен `.catch()` к промису сканирования — защита от необработанных ошибок. При ошибке `scanCompleted` всё равно устанавливается в `true`, и рендереры получают `refresh()`.

## План

1. ✅ Добавить `.catch()` к промису `performCacheScan()`
2. ✅ Перенести сканирование в `onLayoutReady()`
3. Протестировать: перезагрузить Obsidian, убедиться, что таблицы отображают карточки сразу.
