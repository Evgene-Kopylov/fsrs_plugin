## Функция date_format для SELECT

### Контекст

Сейчас поле `due` отображается в формате `YYYY-MM-DD_HH:MM` (Obsidian).
Пользователь хочет управлять форматом даты через SQL-подобный синтаксис:

```sql
SELECT date_format(due, '%Y-%m-%d') AS "Следующее"
WHERE state = "review"
ORDER BY due ASC
LIMIT 20
```

Аналог: `strftime` в SQLite, `DATE_FORMAT` в MySQL.

### Спецификаторы формата

| Спецификатор | Описание | Пример |
|-------------|----------|--------|
| `%Y` | Год (4 цифры) | `2025` |
| `%m` | Месяц (01-12) | `01` |
| `%d` | День (01-31) | `15` |
| `%H` | Часы (00-23) | `14` |
| `%M` | Минуты (00-59) | `30` |

### Тестовые случаи

Дано: карточка с `due = "2025-03-07_14:30"`

- `date_format(due, '%Y-%m-%d')` → `2025-03-07`
- `date_format(due, '%d.%m.%Y')` → `07.03.2025`
- `date_format(due, '%H:%M')` → `14:30`
- `date_format(due, '%Y')` → `2025`
- `date_format(due, '%d %m %Y, %H:%M')` → `07 03 2025, 14:30`
- `date_format(due, '%Y-%m-%d') AS "Дата"` → колонка с заголовком «Дата», значение `2025-03-07`
- `SELECT file, date_format(due, '%d.%m.%Y') AS "След."` → полный запрос с алиасом
- `date_format(reps, '%Y')` → reps не дата, ошибка
- `date_format(due, '%Q')` → неизвестный спецификатор, ошибка
