# Тепловая карта (Heatmap)

Компонент мотивации — показывает сколько повторений сделано по дням.

## Идея

Классический виджет «activity heatmap», как на GitHub: сетка дней, цвет зависит от количества повторений в этот день.

## Требования

- Отображение в виде отдельного блока `fsrs-heatmap`
- Каждый день — ячейка, цвет = интенсивность (0 повторений → нет закраски, много → яркий цвет)
- Данные из WASM-кэша (даты повторений по всем карточкам)
- Поддержка локализации (месяцы, дни недели, тултипы)

## Архитектура (как у fsrs-table)

### Rust/WASM — все вычисления и строки

Метод `get_heatmap_data(now_iso, weeks, locale) → JSON`:
- Проходит по кэшу, считает повторения по датам
- Строит массив ячеек: для каждого дня в диапазоне — дата, количество, уровень цвета (0-4), флаг будущего, готовый текст тултипа
- Вычисляет позиции месяцев для подписей
- Возвращает локализованные названия месяцев и дней недели
- Возвращает заголовок блока
- **Всё** в одном JSON — TS не думает, только рендерит

### TypeScript — только DOM

- `FsrsHeatmapRenderer` (по аналогии с `FsrsTableRenderer`)
- Вызывает `cache.getHeatmapData(now, weeks)`
- В цикле создаёт `div` с готовыми классами, атрибутами, текстами из JSON
- Тултип: `div` с готовым текстом из ячейки
- Никакой логики: без вычислений дат, без pluralization, без выбора строк по локали
