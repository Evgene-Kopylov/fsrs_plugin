## Задача 0.2. Рефакторинг больших файлов

**Приоритет:** низкий (выполняется только если файлы превысили лимит)

**Проблема:**  
Некоторые файлы проекта могут разрастись и превысить 350 строк, что усложняет их поддержку.

**Критерий:** Если файл превышает 350 строк — разбить на модули по смыслу.

**Потенциально крупные файлы:**
- `src/utils/fsrs/incremental-cache.ts` (~250 строк)
- `src/ui/fsrs-table-renderer.ts` (~300 строк)
- `src/utils/fsrs-table-generator.ts` (~150 строк)

**Действия (если файл разросся):**
1. Вынести парсинг SQL в отдельный модуль (уже есть `fsrs-table-params.ts` — проверено).
2. Вынести рендеринг строк таблицы в отдельную функцию вне класса.
3. В `incremental-cache.ts` вынести логику чанков в отдельный метод.

**Коммит:** `refactor: разбиение [имя файла] на модули`

**Выполнять:** отдельным коммитом **до** добавления новой логики, если какой-то файл превысил лимит.
