# WHERE file — совпадение по имени, а не полному пути

## Проблема

`WHERE file = "имя"` делает точное сравнение с полным путём (`папка/имя.md`). Пользователь ожидает что фильтр сработает по имени файла без расширения и пути.

См. инцидент: [where-file-exact-match.md](../incidents/where-file-exact-match.md)

## Решение

В evaluator.rs при сравнении поля `file`:
- Извлекать из значения имя файла без расширения (последний компонент пути, без `.md`)
- Сравнивать с очищенным значением из запроса

Либо:
- В `compute_fields_from_state` добавить поле `filename` — только имя без пути и расширения
- Использовать его для сравнений вместо `file`

## Второй вариант: поддержка LIKE

Добавить оператор `LIKE` для частичного совпадения по строке — решит проблему универсальнее.

## Что выбрать

Добавить поле `filename` в вычисляемые поля. Поле `file` остаётся полным путём для обратной совместимости.

Частичное совпадение по строке — отдельная таска: [like-operator.md](like-operator.md)
