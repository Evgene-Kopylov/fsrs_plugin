# Ленивое построение попапов в хитмапе

## Гипотеза

Построение хитмапа (`renderGrid`) происходит медленно, потому что всё делается синхронно в одном кадре: и сетка, и все DOM-обработчики для попапов (до 1113 `addEventListener`).

## Что сделано

Замена индивидуальных обработчиков на делегирование:
- Убрал `mouseenter`/`mouseleave`/`click` с ячеек
- Добавил `data-idx` на ячейки
- Повесил 3 делегированных обработчика на контейнер (`mouseover`, `mouseout`, `click`) с `closest(".fsrs-heatmap-cell")`

## Результат

**Эффекта по производительности нет.** 1113 `addEventListener` — не bottleneck. Тормозит скорее WASM-сериализация (`getHeatmapData`) или синхронное создание 400+ DOM-узлов.

Изменения откатил — текущий код с индивидуальными обработчиками проще и понятнее.

## Вывод

Если заниматься ускорением хитмапа — копать в сторону WASM (не гонять `reviews` для всех ячеек, запрашивать лениво при hover) либо рендерить сетку чанками через `requestAnimationFrame`.
