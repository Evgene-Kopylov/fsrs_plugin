# Ленивое построение попапов в хитмапе

## Проблема

Построение хитмапа (`renderGrid`) происходит медленно, потому что всё делается синхронно в одном кадре: и сетка, и все DOM-обработчики для попапов. Пользователь ждёт появления всей сетки.

При этом мгновенная отзывчивость попапа на hover — это хорошо, надо сохранить.

## Что нужно

Разделить отображение хитмапа и подготовку попапов:

1. **Фаза 1** — рендер сетки (ячейки, цвета, подписи месяцев/дней). Пользователь видит хитмап как можно раньше. Без обработчиков попапов.
2. **Фаза 2** — ленивая подготовка попапов: навешивание обработчиков и построение DOM попапа — только при hover, не раньше.

Попап при hover/клике всё так же мгновенный и полный, как сейчас.

### Требования

- `buildPopup()` не меняется.
- Мгновенная отзывчивость попапа сохраняется.
- Сетка становится видимой быстрее, чем сейчас.

### Файлы

- `src/ui/fsrs-heatmap-renderer.ts`

### Критерий готовности

- Хитмап появляется без ощутимой задержки.
- Попап при наведении мгновенный, как сейчас.
