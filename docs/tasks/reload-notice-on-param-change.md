# Notice перезагрузки при изменении FSRS-параметров

**Суть:** Если пользователь меняет параметры FSRS в настройках, кэшированные состояния карточек становятся неактуальными. Вместо точечной очистки кэша (`this.cache?.clear()`) — показать notice с предложением перезагрузить плагин.

### Параметры, требующие перезагрузки

- `enable_fuzz` — случайное изменение интервалов
- `request_retention` — целевой уровень запоминания
- `maximum_interval` — максимальный интервал в днях
- `default_initial_stability` — начальная стабильность для новых карточек
- `default_initial_difficulty` — начальная сложность для новых карточек

## Требования

- Проверять в `saveSettings`, изменился ли любой из параметров выше.
- Если да — убрать `this.cache?.clear()`, показать `notice` с сообщением вроде «Настройки FSRS изменены. Перезагрузите плагин для применения».
- Если не изменились — просто сохранить без уведомления.
- Остальные настройки (`ignore_patterns`, `auto_add_review_button`, `status_bar_icon` и т.д.) — просто сохранять как есть, без перезагрузки.

**Порядок добавления:** начать с `enable_fuzz`, остальные — по мере необходимости.
