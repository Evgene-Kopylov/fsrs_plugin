# 为什么插件叫这个名字

最初只是一个简单的间隔重复脚本。
名字不太有创意 — 就叫 SRS。它运行得还不错，
但几周后我开始思考：能不能做得更好？
这方面有没有什么研究？

深入研究后，我发现 Obsidian 中没有适合我的插件。
现有的那些看起来很奇怪 —
过于死记硬背，使用复杂。
问答卡片……给自己的笔记写问题，给自己？
这看起来很奇怪。

然后我发现了 FSRS — 一种现代间隔重复算法。
更令我高兴的是，参考实现是用 Rust 写的。

「我需要的插件不存在」，我想……然后很高兴。
这意味着我可以自己做一个。

项目自然而然地被命名为
FSRS Obsidian plugin。

后来，当插件已经完成时，我才知道 Obsidian 团队
不允许在插件名称中使用「Obsidian」和
「Plugin」这两个词。

于是，由于一连串的事件，插件名称缩短为
简单的 **FSRS** — 恰好与算法名称重合。
*就这样。*
