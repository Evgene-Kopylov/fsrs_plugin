## Спецификация полей для WHERE

### Числовые поля (поддерживают операторы `>`, `<`, `>=`, `<=`, `=`, `!=`):
- **`reps`** (`u32`) - количество повторений карточки
- **`overdue`** (`f64`) - часы просрочки (≥ 0.0, вычисляется как `max(0, now - due_date) / 3600000`)
- **`stability`** (`f64`) - стабильность карточки (0.0–∞)
- **`difficulty`** (`f64`) - сложность карточки (0.0–1.0)
- **`retrievability`** (`f64`) - извлекаемость (0.0–1.0)
- **`elapsed`** (`f64`) - прошло дней с последнего повторения
- **`scheduled`** (`f64`) - запланировано дней до следующего повторения

### Строковые поля (планируются в Фазе 3):
- **`file`** (`String`) - имя файла (поддерживает `=`, `!=`, `LIKE` с wildcards `%` и `_`)
- **`state`** (`String`) - состояние карточки: "New", "Learning", "Review", "Relearning", "due" (поддерживает `=`, `!=`)
- **`due`** (`String`) - дата следующего повторения в формате Obsidian `ГГГГ-ММ-ДД_чч:мм` (сравнение как строки)

### Типы значений:
- **Числа**: целые (`42`) или с плавающей точкой (`3.14`, `0.5`)
- **Строки**: в двойных кавычках (`"значение"`)
- **Даты**: как строки в формате Obsidian (`"2024-01-15_14:30"`)

### Операторы по фазам:
- **Фаза 1**: `>`, `<`, `>=`, `<=`, `=`, `!=` (только для числовых полей)
- **Фаза 2**: `AND`, `OR` (логические операторы)
- **Фаза 3**: `LIKE` (для поля `file`), скобки `( )` для группировки

## Примечания

1. **Обратная совместимость**: Существующие запросы без WHERE должны работать без изменений
2. **Обработка ошибок**: Парсер должен продолжать работу с предупреждениями при некорректных WHERE условиях
3. **Производительность**: WHERE условия применяются перед сортировкой для уменьшения количества обрабатываемых карточек
4. **Типы данных**: 
   - Числовые поля:
     - `reps` (u32) - количество повторений
     - `overdue` (f64) - часы просрочки
     - `stability` (f64) - стабильность
     - `difficulty` (f64) - сложность
     - `retrievability` (f64) - извлекаемость
     - `elapsed` (f64) - прошло дней
     - `scheduled` (f64) - запланировано дней
     Все поддерживают операторы `>`, `<`, `>=`, `<=`, `=`, `!=`
   - Строковые поля:
     - `file` (только `=`, `!=`, `LIKE`)
     - `state` (только `=`, `!=`)
     - `due` (только `=`, `!=`, `>`, `<`, `>=`, `<=` как строки в формате Obsidian)
5. **Оператор LIKE**: Поддержка wildcards: `%` - любое количество символов, `_` - один символ

## Текущие ограничения

1. **Глубина вложенности**: Скобки поддерживаются на одном уровне вложенности (для упрощения)
2. **Типы операторов**: 
   - `LIKE` доступен только для поля `file` (в Фазе 3)
   - Для поля `due` сравнение как строк в формате Obsidian (ГГГГ-ММ-ДД_чч:мм)
3. **Логические операторы**: `NOT` не поддерживается в первой фазе
4. **Регистр**: Ключевые слова регистронезависимы, значения строк - регистрозависимы для `LIKE`
5. **Типы значений**: Числа могут быть целыми или с плавающей точкой, строки в двойных кавычках
