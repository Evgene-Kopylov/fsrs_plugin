## Сброс отступов в wasm-review.ts

**Проблема:** При редактировании `src/utils/fsrs/wasm-review.ts` через `edit_file` табы были заменены на пробелы. Попытка восстановить через heredoc в `terminal` провалилась из-за ошибки синтаксиса оболочки.

**Статус:** Файл был перезаписан с пробелами. На момент инцидента исходная кодировка — табы. Решение не найдено — суб-агент прерван пользователем.

**Рекомендация:** Для файлов с табами использовать `edit_file` в режиме `edit`, а не `overwrite`, и проверять отступы после записи.

## 2026-04-26: edit_file отказывается редактировать файл («No edits were made»)

**Проблема:** После нескольких успешных `edit_file` подряд в один и тот же файл (`fsrs-table-filter.ts`), последующие попытки редактирования возвращают «No edits were made» без объяснения причины. Файл не заблокирован — `restore_file_from_disk` показывает «clean». 

**Решение:** Прочитать файл через `read_file`, после чего повторная попытка `edit_file` проходит успешно.

**Причина (предположительно):** Инструмент `edit_file` кэширует состояние файла на момент последнего чтения. После нескольких последовательных правок внутреннее представление расходится с реальным содержанием файла, и инструмент не может применить очередное изменение. Чтение свежей версии через `read_file` сбрасывает кэш.

**Признак:** Первая и вторая правки проходят, третья и последующие — «No edits were made» без ошибки в консоли.

**Что делать, если edit_file молча отказывается:**
1. Прочитать файл через `read_file` (любую часть).
2. Повторить `edit_file` с тем же описанием — должен сработать.
3. Если не помогло — подождать 2-4 секунды и повторить (возможна блокировка ОС).

**Статус:** Воспроизводится нестабильно. Лечится чтением файла.
